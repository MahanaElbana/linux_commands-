# What is Soft Link And Hard Link In Linux? üî≠
 
 - **A symbolic or soft link** is an actual link to the original file üèÜ
 - whereas **a hard link** is a mirror copy of the original file üèÜ 
 - Any change in **the Origin life** affects **the hard or soft link** üèÜ
 - Any change in **the hard or soft link** affects **the Origin life** üèÜ
 
 - the difference betwwen *soft and hard link* üèÜ

   - **A symbolic or soft link** üî¶
      - If you delete the original file, the soft link has no value, because it points to a non-existent file.
   
   - **hard link** üî¶
      - But in the case of **hard link**, it is entirely opposite. 
      - Even if you delete the original file, the hard link will still has the data of the original file.
      - Because hard link acts as a mirror copy of the original file.

 - <H2 align="center"> üî¶ Part 1 : Soft Link üî¶ </H2> 
 
 - How to create a soft link ‚òÑÔ∏è 
   ```
   ln -s origin_file_path  linked_file_path_name
   ```  
 - How to create a soft link on an existing file (overwrite) ‚òÑÔ∏è
   ```
   ln -sf origin_file_path  linked_file_path_name
   ```  
 - <H2 align="center"> üî¶ Part 2 : Hard Link üî¶ </H2> 
 
 - How to create a Hard link ‚òÑÔ∏è  
   ```
   ln  origin_file_path  linked_file_path_name
   ```

# Firewall with **UFW** *Uncomplicated Firewall* on Ubuntu üî≠

   - **Defination** : **ufw** is The default firewall configuration tool for Ubuntu .

   - <H2 align="center"> üî¶ Part 1 : enable ufw üî¶ </H2>  
     
     
     ```
     sudo ufw enable
     
     ```
    
   - <H2 align="center"> üî¶ Part 2 : add rule to firewall üî¶ </H2>   

     ```
     sudo ufw allow 22 && sudo ufw allow 80 && sudo ufw allow 8000
     sudo ufw allow 443
     sudo ufw allow http
     sudo ufw allow https
     sudo ufw delete allow 80
     ```

   - <H2 align="center"> üî¶ Part 3 : allow rule  üî¶ </H2>  

   - How to enable **(https)** = **port(443)**  ‚òÑÔ∏è
     ```
     sudo ufw allow https
     sudo ufw allow 443
     ``` 
   - How to enable **(http)** = **port(80)**  ‚òÑÔ∏è
     ```
     sudo ufw allow http
     sudo ufw allow 80
     ```   
   - How to enable **(ssh)**  ‚òÑÔ∏è
     ```
     sudo ufw allow ssh
     ```  
   - How to enable **port(22)**  ‚òÑÔ∏è
     ```
     sudo ufw allow 22
     ```  
   -  <H2 align="center"> üî¶ Part 4 : Deleting a Rule üî¶ </H2>       
        
   - How to delete **port(22)** ‚òÑÔ∏è
     ```
     sudo ufw delete allow 22/tcp
     ``` 
   - How to delete **https** = **port(443)** ‚òÑÔ∏è
     ```
     sudo ufw delete allow https
     sudo ufw delete allow 443

     ```      
   - How to delete **http** **port(80)** ‚òÑÔ∏è
     ```
     sudo ufw delete allow http
     sudo ufw delete allow 80
     ``` 
   -  <H2 align="center"> üî¶ Part 5 : disable ufw  üî¶ </H2>
      ```
      sudo ufw disable
      ```    
   -  <H2 align="center"> üî¶ Note for DigitalOcean üî¶ </H2>

   - important steps to can access on your server again ‚òÑÔ∏è

      -  Once you set up a new Machine on **Digital Ocean** add üëá 
         ```
         sudo ufw enable 
         sudo ufw allow ssh && sudo ufw allow 80 && sudo allow 443
         sudo ufw allow OpenSSH
         sudo ufw allow 20
         sudo ufw status
         sudo ufw disable
         ```  
      - Don't forget to disable **ufw**  ‚ö†Ô∏è

# step zero : How to create group on linux
``` 
groupadd [OPTIONS] GROUPNAME 
```
```
sudo groupadd www-data
```
# step zero :  /home/platrainproject/PlatRain
```
 sudo chown mahney:www-data /media/hardmahney/E_Conclusion/django_gunicorn_example/PlatRain/
```
# step one : Create a Gunicorn systemd Socket File (*gunicorn.socket*)

```
sudo nano /etc/systemd/system/gunicorn.socket
```

```gunicorn.socket
[Unit]
Description=gunicorn socket

[Socket]
ListenStream=/run/gunicorn.sock

[Install]
WantedBy=sockets.target
```

# step two : Create a Gunicorn systemd Service File (*gunicorn.service*)

```
sudo nano /etc/systemd/system/gunicorn.service
```

```
[Unit]
Description=gunicorn daemon
Requires=gunicorn.socket
After=network.target
[Service]
User=mahney
Group=www-data
WorkingDirectory=/media/hardmahney/E_Conclusion/django_gunicorn_example/PlatRain
ExecStart=/media/hardmahney/E_Conclusion/django_gunicorn_example/venv/bin/gunicorn \
          --access-logfile - \
          --workers 3 \
          --bind unix:/run/gunicorn.sock \
          project.wsgi:application
[Install]
WantedBy=multi-user.target
```

# Step three  

```
sudo systemctl start gunicorn
sudo systemctl enable gunicorn
sudo systemctl status gunicorn.socket
```
